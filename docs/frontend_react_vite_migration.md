# Frontend Desktop Shell

Документ фиксирует текущее состояние нового desktop frontend после очистки старой операторской верстки.

## Статус

Первый этап desktop-верстки готов для визуальной проверки:

- sidebar собран по утвержденному desktop-дизайну;
- header закреплен в правой рабочей области и не уезжает при скролле;
- навигация переключает корректные заглушки разделов;
- старые вкладки операторской панели удалены из `frontend/src`;
- route `/operator` больше не отдает старую HTML-операторку, а ведет на `/operator-next`.

## Источник дизайна

Актуальный источник для desktop:

```text
design/Desktop/stitch_infocollect_design_system
```

Для первого этапа используется shell из варианта `infocollect_desktop_refined_4`. Дизайн-исходники не редактируются.

## Структура frontend

- `src/App.jsx` - минимальная оболочка приложения и настройка Ant Design theme.
- `src/shell/DesktopShell.jsx` - UI-каркас desktop.
- `src/shell/desktopNavigation.js` - структура меню и метаданные разделов.
- `src/styles.css` - стили shell.
- `tests/desktop-shell.test.mjs` - smoke-тест порядка меню.

## Sidebar

Основные разделы:

1. Панель управления
2. Реестр оборудования
3. Объекты
4. Инспекции
5. Склад
6. Поступления
7. Расхождения
8. Синхронизация
9. История
10. Отчёты

Администрирование:

1. Сотрудники
2. Роли и права
3. Системные настройки

Этот порядок считается утвержденным для desktop shell.
