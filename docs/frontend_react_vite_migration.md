# React + Vite Migration

В проекте используется отдельный фронтенд-модуль `frontend/` для поэтапного переноса операторской панели на `React + Vite`.
В качестве визуальной и архитектурной базы выбран бесплатный шаблон `larry-xue/react-admin-dashboard` на стеке `React + Vite + Ant Design`.

## Текущий статус

- исходный шаблон без правок хранится в `vendor/react-admin-dashboard-template`
- страница просмотра оригинала шаблона: `http://127.0.0.1:8000/operator-template`
- рабочая адаптируемая операторская панель: `http://127.0.0.1:8000/operator-next`
- мобильный полевой контур продолжает жить отдельно и не зависит от новой операторской панели

## Что уже перенесено

- экран входа и сессионная авторизация
- ролевой доступ:
  - `field_worker` — только свои назначения
  - `operator` — операторская панель без вкладки сотрудников
  - `admin` — полный доступ
- вкладки `Контроль`, `Назначения`, `Сотрудники`, `Группы`, `Экспорт`
- создание, редактирование, отключение и восстановление сотрудников
- загрузка и замена фото сотрудника
- сводки, фильтры, группы и экспортные представления
- отдельный экран входа и обязательная авторизация перед открытием панели
- редактирование собственного профиля через меню пользователя
- переключение светлой/темной темы и выбор акцентной палитры
- адаптивный shell для мобильного режима с drawer-навигацией
- контекст по помещениям в `Назначениях`: состав помещения, последние отметки, кто и когда их внес

## Что изменено в архитектуре фронтенда

- новый интерфейс строится на `Ant Design`, а не на самописном CSS-каркасе
- используется шаблонная схема `Header + Sidebar + Content`
- меню строится динамически по роли пользователя
- бизнес-логика старой панели переносится по модульному принципу в React-компоненты
- оригинальный шаблон сохраняется отдельно как эталон для сравнения и дальнейших визуальных решений

## Логика ролей

### `field_worker`

- видит только собственный кабинет
- работает только со своими назначениями
- не имеет доступа к вкладкам контроля, сотрудников, групп и экспорта

### `operator`

- имеет доступ к операторской панели
- видит вкладки контроля, назначений, групп и экспорта
- не имеет доступа к вкладке сотрудников

### `admin`

- имеет полный доступ ко всей операторской панели
- управляет сотрудниками, ролями, назначениями и остальными разделами

## Что дополнительно проверено

- production-сборка `operator-next` проходит без ошибок
- backend-компиляция `python -m compileall app` проходит без ошибок
- живые маршруты `GET /api/health`, `POST /api/auth/login`, `POST /api/users/field-workers`, `POST /api/users/field-workers/{id}/avatar` отрабатывают успешно
- `/operator-next` отдает актуальный production bundle после сборки

## Почему выбран именно этот путь

- можно сохранить рабочую старую операторку до полного завершения миграции
- можно отдельно сравнивать оригинальный шаблон и адаптированную реализацию
- `Ant Design` лучше подходит для плотных операторских интерфейсов с таблицами, формами, статусами и рабочими списками
- перенос по вкладкам снижает риск сломать весь интерфейс одним изменением

## Как запускать

### Backend

```powershell
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

### Frontend production build

```powershell
cd frontend
node .\node_modules\vite\bin\vite.js build
```

### Адреса

- основная новая панель: `http://127.0.0.1:8000/operator-next`
- оригинальный шаблон: `http://127.0.0.1:8000/operator-template`
