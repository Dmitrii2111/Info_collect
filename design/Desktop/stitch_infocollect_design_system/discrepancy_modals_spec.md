# Discrepancy Resolution Modals Reference

## 1. Confirm: Accept Operator Data
**Title:** Принять данные оператора?
**Message:** Плановые данные будут заменены данными, которые оператор зафиксировал во время инспекции.
**Context:**
- Инспекция: #INS-2024-001
- Помещение: 2.01.30 — Кабинет осмотра
- Оборудование: Система клапанная
- Оператор: Иван Иванов
**Comparison:**
- Плановые данные: SN-90831 (на месте по плану)
- Данные оператора: SN-90381 (найдено оператором)
**Required Field:** Комментарий решения (Placeholder: Укажите причину принятия данных оператора)
**Impact:** 
- Данные оператора сохраняются
- Статус -> Разрешено
- Решение пишется в историю
- Инспекция разблокируется
**Footer:** [Отмена] [Принять данные оператора (Blue)]

## 2. Confirm: Keep Planned Data
**Title:** Оставить плановые данные?
**Message:** Данные оператора будут отклонены для этого расхождения.
**Context:** Same as above
**Required Field:** Комментарий решения (Placeholder: Укажите причину сохранения плановых данных)
**Impact:** 
- Плановые данные остаются
- Данные оператора отклоняются
- Статус -> Разрешено
- Решение пишется в историю
**Footer:** [Отмена] [Оставить плановые данные (Blue)]

## 3. Confirm: Close Discrepancy (Destructive)
**Title:** Закрыть расхождение?
**Message:** Расхождение будет закрыто без изменения плановых данных. Используйте, если проблема не актуальна.
**Context:** #INS-2024-001 • Конфликт • Система клапанная
**Required Field:** Причина закрытия (Placeholder: Укажите причину закрытия расхождения)
**Impact:** 
- Перенос в архив
- Данные не меняются
- Запись в историю
**Footer:** [Отмена] [Закрыть расхождение (Red)]

## 4. Success State
**Title:** Расхождение разрешено
**Message:** Решение сохранено в истории.
**Footer:** [Закрыть] [К списку расхождений]

## 5. Error State
**Title:** Не удалось разрешить расхождение
**Message:** Система не смогла применить выбранное решение.
**Reason:** Сервер временно недоступен.
**Footer:** [Закрыть] [Открыть синхронизацию] [Повторить (Blue)]
